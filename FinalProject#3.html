What is a Raspberry Pi?

Its a fully featured, internet connected Linux computer that is easy to embed into prototypes (in other words, a low cost, credit card sized computer)

It's also a platform for interactive devices and the Internet of Things (GPIO pins and HAT extensions make it easy to add sensors, physical controls, outputs and interfaces to your projects).

Raspberry Pi vs. Arduino, what gives?


Rpi - Microprocessor

Good for processing data
Linux OS
HDMI/ Audio output/ USB / Wifi/ Bluetooth
Only digital GPIO pins
High power consumption - 750mW+


Arduino - Microcontroller


Good at controlling inputs/outputs
Only runs the code you put on it. No OS or firmware
Digital and Analog GPIO pins
Low power consumption - 175mW


Choose Arduino for:

Controlling lots of sensors, small LCDs and motors, low power applications (ie. battery powered projects) and analog sensors.


Choose Raspberry Pi for:

Handling video or graphic interfaces, projects that process lots of data ( ie. data visualization ) and computer vision and other performance-dependant processes.

What you will need:

Attach UBS/Power Last

Just like with Arduino, we want to make sure that our circuits are complete before powering. With Raspberry Pi, this also ensures all peripherals initialize properly while it boots up. Make sure your Pi is off while making changes to attached circuits.



STEP #1

Install the Micro SD card in the back.




STEP #2

Plug in the HDMI.


STEP #3

Plug in a USB Mouse.



STEP #4

Plug in a USB Keyboard.



STEP #5

Plug in the Power adapter to Micro USB.

Raspbian Install

This screen (NOOBS) allows us to choose an operating system to install. These cards come preinstalled, but a fresh setup would require moving some files onto the SD card. Choose Raspbian and click Install.




One Last Setup Set

The setup wizard will walk you though setting location (for time), keyboard type (usually US layout), and changing the system password. These can all be changed later, through the “Raspberry Pi Configuration” menu item.

You’re already familiar with the Arduino IDE

The Arduino programming language is based on C, and used the Arduino IDE as the environment to program and run code.



Thonny is similar for Python

We’ll be using the Python programming language in the Thonny IDE. It’s very light on syntax ( easy for humans to read and write ), and interfaces well with RPi peripherals like SenseHAT.



Python vs. Processing/Arduino


Python

Interpreted, generally runs slower
Typically less lines of code, and more human readable
Indentation is part of the code, incorrect spacing will cause errors
Dynamically typed variables, decided at run time
Not tied to a specific IDE ( VS Code is a good one )
No built in looping


Processing/Arduino

Compiled, generally runs faster
More syntax
Indentation is purely visual, but a good practice
Statically typed variables (ie. int)
Bundled with an IDE
Have a built in looping function ( loop() or draw() )


Indentation

In Python, indentation is part of the code, replacing brackets. This means it will cause errors if incorrect. Can use either spaces or tabs, but it’s best to pick one.



Dynamic typing

Variables don’t need to be set to a specific type of data, and can change at runtime.
